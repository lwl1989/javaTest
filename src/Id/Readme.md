Identity

Id核心意义： 唯一性

最好保证 趋势自增性（提高性能,参照btree+为例）

海量数据存贮优化（分库分区分表）

数据库连接数 有限 （分库  提高连接数）

单表性能瓶颈 （分表 减低锁粒度，降低表性能【参考map reduce思想】）

常用分表算法  

     取模： 优势简单  劣势： 不好扩容 （常见 crc32 hash取模）
     
     hash: 好扩容  分布不均
     
     位移:  uid >> seed
     
一般都是通过自增id来进行处理


一些特殊业务处理：

单调递增， 信息安全（不能用id来暴露）, 信息冗余（通过id就能获取一些更多信息，比如多位id组合  比如：业务号+时间+业务表+实际ID）
